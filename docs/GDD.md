# Game Design Document (GDD)
## Word Car Game — 초등 영어 단어 런닝 게임

**버전**: 1.2
**작성일**: 2026-06-12
**대상**: 초등학교 1~6학년

---

## 1. 게임 개요

### 1.1 컨셉 한줄 요약
> 달리는 자동차로 장애물을 피하면서 영어 단어를 학습하는 사이드스크롤 런닝 게임

### 1.2 장르 & 레퍼런스
- **장르**: 엔드리스 런너 (Endless Runner) + 어휘 퀴즈
- **레퍼런스**: 구글 공룡 게임 (Chrome Dino), 펭귄 점프 게임
- **차별점**: 단순 점수 경쟁이 아닌, 단어 퀴즈를 통해 스테이지를 클리어하는 학습 게임

### 1.3 플레이 흐름

```
메인 메뉴
    ↓ [게임 시작]
스테이지 N 시작 (배너 표시)
    ↓ [자동 주행 + 장애물 회피]
거리 게이지 45% 도달
    ↓ [⭐ 단어 아이템 등장]
자동차가 아이템에 닿으면 → 퀴즈 팝업 (물리 일시정지)
    ↓
  [정답]                    [오답]
    ↓                          ↓
  "목표 지점까지 달리세요"    목숨 -1, 스테이지 재시작
    ↓ [계속 주행 + 장애물 회피]
거리 게이지 100% 도달
    ↓
STAGE CLEAR → 스테이지 N+1
    ↓ (반복)
게임 오버 (목숨 0)
    ↓
점수 등록 → 하이스코어
```

## 2. 게임 메카닉

### 2.1 플레이어 (자동차 🚗)

| 속성 | 값 |
|------|-----|
| 이동 방향 | 자동 스크롤 (배경이 좌→우로 움직임) |
| 점프 입력 | Space / ↑ 방향키 / 화면 우측 탭 |
| 앞뒤 이동 | ← / → 또는 A / D 키, 화면 좌측 ◀▶ 버튼 |
| 이모지 방향 | 이동 방향에 따라 자동 전환 (← 입력 시 좌향, → 입력 시 우향) |
| 점프 높이 | 고정 (초기 속도 −500px/s, 중력 1150px/s²) |
| 더블 점프 | 없음 |
| 목숨 | 3개 ❤️❤️❤️ |
| 무적 시간 | 충돌 후 1.5초 |

**조작 구역 구분:**
- 화면 x > 155 탭: 점프
- 화면 x ≤ 155 하단 ◀▶ 버튼: 좌우 이동

### 2.2 장애물

| 장애물 | 이모지 | 특성 |
|--------|--------|------|
| 구덩이 | 🕳️ | 낮은 장애물 |
| 바위 | 🪨 | 중간 높이 장애물 |
| 선인장 | 🌵 | 높은 장애물, 점프 필수 |
| 이중 선인장 | 🌵🌵 | 넓은 장애물, 스테이지 4+ 등장 |

**스폰 규칙:**
- 스테이지 **전체 구간**에서 지속 생성 (단어 아이템이 화면에 있어도 멈추지 않음)
- 생성 간격 공식: `Math.max(600, 1000 − (stage − 1) × 20)` ms
  - 스테이지 1: **1000ms** (1초마다)
  - 스테이지 20: 620ms
  - 최소 간격: **600ms**
- 스테이지 4 미만: 구덩이 / 바위 2종만 등장
- 스테이지 4+: 선인장 / 이중선인장 추가

### 2.3 단어 아이템 (⭐)

- **등장 시점**: 거리 게이지 **45%** 도달 시 화면 우측(x=850)에서 등장
- **이동**: 스크롤 속도와 동일하게 좌→우 이동
- **Y 위치**: 도로 표면 약 55px 위 (y ≈ groundY − 55 = 275)
- **수집 방법**: 자동차가 아이템에 닿으면 자동 수집
  - 지면을 달리면서 수집 가능 (점프 불필요)
  - 낮은 점프 중에도 수집 가능
  - 높은 점프(최대 높이 근처)에서는 자동차가 아이템 위로 지나가므로 수집 안 됨
- **미수집 시**: 화면 왼쪽 끝(x < −60) 이탈 시 오른쪽에서 재등장 (반복)
- **수집 후**: 퀴즈 팝업 트리거, 물리 일시정지
- **스테이지 클리어 조건**: 퀴즈 정답 **AND** 거리 게이지 100% 도달 (두 조건 동시 충족)
- **시각 효과**: scale + rotation tween, repeat −1 (반짝임)

### 2.4 퀴즈 시스템

**퀴즈 규칙:**
- 힌트: 이모지 그림 + 영어 정의 1줄
- 선택지: 4개 (정답 1개 + 오답 3개, 무작위 배치)
- 제한시간: 15초
- 시간 초과 = 오답 처리

**정오답 처리:**

| 상황 | 결과 |
|------|------|
| 정답 | `quizPassed = true`, 달리기 재개, "목표 지점까지 달리세요" 메시지 |
| 정답 후 100% 도달 | `completeStage()` 호출, STAGE CLEAR |
| 빠른 정답 (5초 이내) | 스테이지 클리어 시 +50점 추가 |
| 오답 | 목숨 −1, 스테이지 재시작, 점수 → 스테이지 시작 점수로 초기화 |
| 시간 초과 | 오답과 동일 처리 |

## 3. 스테이지 설계

### 3.1 스테이지 구조

각 스테이지 = **거리 목표 도달 + 단어 퀴즈 정답** (두 조건 모두 필요)

| 단계 | 스테이지 | 학년 | 게임 속도 | 목표 거리 |
|------|---------|------|---------|-----------|
| 초급 | 1~5 | 1~2학년 | 190~238 px/s | 1900~2260px |
| 중급 | 6~10 | 3~4학년 | 250~298 px/s | 2350~2710px |
| 고급 | 11~15 | 5~6학년 | 310~358 px/s | 2800~3160px |
| 무한 | 16+ | 혼합 | 최대 440 px/s | 최대 4200px |

**속도 공식**: `Math.min(440, 190 + (stage − 1) × 12)` px/s
**거리 공식**: `Math.min(4200, 1900 + (stage − 1) × 90)` px

### 3.2 학년 배정

| 스테이지 | 학년 |
|---------|------|
| 1, 3, 5 | 1학년 |
| 2, 4 | 2학년 |
| 6, 8, 10 | 4학년 |
| 7, 9 | 3학년 |
| 11, 13, 15 | 5학년 |
| 12, 14 | 6학년 |
| 16+ | 1~6학년 무작위 |

### 3.3 배경 변화

| 스테이지 | 배경 테마 |
|---------|---------|
| 1~5 | 낮 (파란 하늘 + 초록 들판) |
| 6~10 | 저녁 (주황/노을) |
| 11~15 | 밤 (검은 하늘 + 별) |
| 16+ | 우주 (검정 + 별) |

## 4. 점수 시스템

### 4.1 점수 계산

| 항목 | 점수 |
|------|------|
| 이동 거리 | +1점 / 10px |
| 퀴즈 정답 + 스테이지 클리어 | +150점 × 콤보 배율 |
| 빠른 정답 (5초 이내) | +50점 추가 |
| 연속 정답 2회 | ×1.5 배율 |
| 연속 정답 3회+ | ×2.0 배율 (최대) |
| 오답 시 점수 | 해당 스테이지 시작 점수로 초기화 |

**콤보 계산 예시:**
- 1연속 정답: 150 × 1.0 = **150점**
- 2연속 정답: 150 × 1.5 = **225점**
- 3연속 이상: 150 × 2.0 = **300점**
- 5초 이내 빠른 정답 추가: **+50점**

### 4.2 하이스코어 저장

- **저장 위치**: localStorage (키: `wordCarGame_scores`)
- **저장 항목**: `{ name, score, stage, date }`
- **최대 저장**: Top 10
- **입력**: 게임 오버 시 이름 입력 (최대 10자)

## 5. UI/UX 설계

### 5.1 HUD 구성 (캔버스 상단)

| 위치 | 내용 |
|------|------|
| 좌측 (x=18) | STAGE N \| GRADE N |
| 중앙 (x=220) | SCORE N  COMBO xN |
| 우측 (x=610) | LIFE ❤️❤️❤️ |
| 중앙 상단 | 노란색 진행 게이지 (목표 거리 진척도) |

**노란 게이지 의미:**
- 0% → 45%: 달리기 + 장애물 회피 구간
- 45%: ⭐ 단어 아이템 등장
- 45% → 100%: 퀴즈 후 달리기 구간 (퀴즈 통과 상태에서 게이지 채우면 클리어)
- 100% + 퀴즈 통과: **STAGE CLEAR**

### 5.2 반응형 / 모바일

- 가로 모드(landscape) 권장
- 캔버스: 800×400 기준, `Phaser.Scale.FIT` + `CENTER_BOTH`
- 터치 조작:
  - 화면 우측(x > 155) 탭: 점프
  - ◀ 버튼 (x=50, y=280): 후진
  - ▶ 버튼 (x=120, y=280): 전진
