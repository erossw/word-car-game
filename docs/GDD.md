# Game Design Document (GDD)
## Word Car Game — 초등 영어 단어 런닝 게임

**버전**: 1.0  
**작성일**: 2026-06-11  
**대상**: 초등학교 1~6학년

---

## 1. 게임 개요

### 1.1 컨셉 한줄 요약
> 달리는 자동차로 장애물을 피하면서 영어 단어를 학습하는 사이드스크롤 런닝 게임

### 1.2 장르 & 레퍼런스
- **장르**: 엔드리스 런너 (Endless Runner) + 어휘 퀴즈
- **레퍼런스**: 구글 공룡 게임 (Chrome Dino), 펭귄 점프 게임
- **차별점**: 단순 점수 경쟁이 아닌, 단어 퀴즈를 통해 스테이지를 클리어하는 학습 게임

### 1.3 플레이 흐름
```
메인 메뉴
    ↓ [게임 시작]
스테이지 N 시작
    ↓ [자동 주행]
장애물 회피 (점프)  ←→  거리 누적 + 점수 증가
    ↓ [⭐ 아이템 수집]
퀴즈 팝업 (게임 일시 정지)
    ↓
  [정답]            [오답]
    ↓                  ↓
  +100점            목숨 -1
    ↓                  ↓
스테이지 클리어    목숨 0이면 게임 오버
    ↓
스테이지 N+1 시작
    ↓ (반복)
게임 오버
    ↓
점수 등록 (이름 입력)
    ↓
하이스코어 화면
```

## 2. 게임 메카닉

### 2.1 플레이어 (자동차 🚗)

| 속성 | 값 |
|------|-----|
| 이동 방향 | 자동 (좌→우 스크롤) |
| 점프 입력 | Space / ↑ 방향키 / 화면 탭 |
| 점프 높이 | 고정 (장애물 높이 기준) |
| 더블 점프 | 없음 |
| 목숨 | 3개 ❤️❤️❤️ |
| 무적 시간 | 충돌 후 1.5초 |

### 2.2 장애물

| 장애물 | 이모지 | 특성 |
|--------|--------|------|
| 구덩이 | 🕳️ | 바닥 없는 구간, 점프로 통과 |
| 바위 | 🪨 | 도로 위 장애물, 점프로 통과 |
| 선인장 | 🌵 | 도로 위 장애물, 높이 변형 가능 |
| 이중 선인장 | 🌵🌵 | 넓은 장애물, 높은 점프 필요 |

**스폰 규칙:**
- 최소 간격: 스테이지 속도에 따라 조정 (충분한 반응 시간 보장)
- 스테이지가 올라갈수록 빈도 증가, 복합 패턴 등장

### 2.3 단어 아이템 (⭐)

- 일정 거리마다 1개 등장 (스테이지당 1회 반드시 등장)
- 자동차가 아이템에 닿으면 수집 → 퀴즈 팝업 트리거
- 아이템 수집 전까지는 스테이지 클리어 불가
- 시각 효과: 반짝임 애니메이션, 수집 시 파티클

### 2.4 퀴즈 시스템

**퀴즈 규칙:**
- 힌트: 이모지 그림 + 영어 정의 1줄
- 선택지: 4개 (정답 1개 + 오답 3개, 무작위 배치)
- 제한시간: 15초
- 시간 초과 = 오답 처리 (목숨 -1)

**정오답 처리:**

| 상황 | 결과 |
|------|------|
| 정답 (15초 이내) | +100점, 스테이지 클리어 |
| 빠른 정답 (5초 이내) | +100점 + 빠른 답변 보너스 +50점 |
| 오답 | 목숨 -1, 정답 표시 후 스테이지 재시작 |
| 시간 초과 | 오답과 동일 처리 |

## 3. 스테이지 설계

### 3.1 스테이지 구조

각 스테이지 = **거리 목표 + 단어 퀴즈 1회**

| 단계 | 스테이지 | 학년 | 게임 속도 | 장애물 빈도 |
|------|---------|------|---------|-----------|
| 초급 | 1~5 | 1~2학년 | 느림 | 낮음 |
| 중급 | 6~10 | 3~4학년 | 보통 | 보통 |
| 고급 | 11~15 | 5~6학년 | 빠름 | 높음 |
| 무한 | 16+ | 혼합 | 점점 빨라짐 | 계속 증가 |

### 3.2 배경 변화

| 스테이지 | 배경 테마 |
|---------|---------|
| 1~5 | 낮 (파란 하늘 + 초록 들판) |
| 6~10 | 저녁 (주황/노을) |
| 11~15 | 밤 (검은 하늘 + 별) |
| 16+ | 우주 (검정 + 별 스크롤) |

---

## 4. 점수 시스템

### 4.1 점수 계산

| 항목 | 점수 |
|------|------|
| 이동 거리 | 1점 / 10px |
| 스테이지 클리어 | +50점 |
| 퀴즈 정답 | +100점 |
| 빠른 정답 (5초 이내) | 추가 +50점 |
| 연속 정답 콤보 (2연속) | ×1.5 배율 |
| 연속 정답 콤보 (3연속+) | ×2.0 배율 (최대) |

### 4.2 하이스코어 저장

- **저장 위치**: localStorage (키: `wordCarGame_scores`)
- **저장 항목**: `{ name, score, stage, date }`
- **최대 저장**: Top 10
- **입력**: 게임 오버 시 이름 입력 (최대 10자)

## 5. UI/UX 설계

### 5.1 반응형 / 모바일

- 가로 모드(landscape) 권장
- 터치: 화면 탭 = 점프, 퀴즈 버튼 클릭
- 캔버스 크기: 800×400 기준, 화면 크기에 맞게 scale
